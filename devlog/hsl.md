# HSL 은 색을 조정하기 좋은 형태로 바꿔준다.

RGB 에서의 색을 어둡게 만드는 방법
{255, 80 , 40} -> 각각 어둡게 해야함. 하지만 이 방식은 저번에도 설명했듯 색감이 탁해지거나 hue값이 틀어질 수 있음.

반면 HSL에서는

```
{
    h: 12 // 색상
    s: 1.0 // 채도: 선명함
    l: 0.58 // 밝기
}
```

여기서 색상은 유지하고 채도를 조금 낮추고 밝기를 조금 어둡게 변경하면, 조정이 쉽게 됨.
이게 HSL의 장점임.

```
{
    h: 12 // 색상
    s: 0.75 // 채도: 선명함
    l: 0.25 // 밝기
}
```

# LAB와 HSL의 역활차이

```
LAB = 색끼리 비슷한지 판단
HSL = 색을 보기 좋게 조정
```

예를들어

```
빨강 후보 A
빨강 후보 B
주황 후보 C
흰색 후보 D
검정 후보 E
```

가 있을 경우 LAB/Delta E는 이런 질문에 좋음

```
A와 B는 같은 계열로 묶어도 되나?
A와 C는 얼마나 다른가?
```

HSL는 이런 질문에 좋음

```
D는 너무 밝으니까 배경 후보에서 제외할까?
E는 너무 어두우니까 메인 컬러로는 별로인가?
A는 너무 쨍하니까 배경용으로 채도를 낮출까?
C는 배경으로 쓰기 좋게 밝기를 낮출까?
```

-> 둘이 같은 상황이 아닌 역활에 따라 다르게 사용하는 것이 좋음.

# HSL 를 쓰면 좋은 색상을 뽑아낼 수 있는 이유.

원본 후보색상이 너무 밝거나 그런 경우

```
조금 어둡게
조금 덜 쨍하게
blur/gradient로 부드럽게
검은 overlay로 안정감 주기
```

을 할 수 있음

```
조금 어둡게  → lightness 낮추기
조금 덜 쨍하게 → saturation 낮추기
```

# 그래서 HSL이 정확하게 뭔데.

앞에서 말했듯이 HSL는 세가지의 값으로 이뤄져있음.

```
interface HSLColor {
  h: number; // 0 ~ 360
  s: number; // 0 ~ 1
  l: number; // 0 ~ 1
}
```

각각의 의미는

```
h = hue, 색상 각도
s = saturation, 채도
l = lightness, 밝기
```

h부터 예를 들면..

```
빨강: h ≈ 0
노랑: h ≈ 60
초록: h ≈ 120
청록: h ≈ 180
파랑: h ≈ 240
보라: h ≈ 280
```

이고 s의 경우 0에 가까우면 회색/흰색/검정 계열 이거 1에 가까울 수록 선명한 색상이 나오게됨.
밝기 l은 0에 가까우면 검정이고. 1에 에 가까우면 흰색이 됨. 즉 0.5 정도가 가장 선명한 색상이라 인식될 가능성이 높음. ( 너무 밝거나 어두우면 색상이 탁해보이는 효과를 가질 수 있음. 그리고 착시가 일어날 수 도 있음. )

# 변환과정

## 먼저 RGB를 0~1으로 변경함.

```
const red = color.r / 255;
const green = color.g / 255;
const blue = color.b / 255;
```

RGB는 0~255임. HSL계산에서는 보통 0~1범위로 정규화 한 후 계산함 이유는 계산 편의성에 있음.

## max / min 을 구함.

```
const max = Math.max(red, green, blue);
const min = Math.min(red, green, blue);
```

이 두값으로 밝기와 채도를 계산함.
max 는 가장 강한, min은 가장 약한 밝기를 가진 채널임.

예를 들어 빨강인 경우

```
red = 1, green = 0, blue = 0, max = 1 , min = 0
```

## Lightness

```
const lightness = (max + min) / 2;
```

HSL의 밝기는 가장 밝은 채널과 가장 어두운 채널의 중간값임.

그래서 rgb 의 경우 255 , 0 , 0 의 빨강은 max가 1이 나오고 min이 0 그래서 0.5가 나오게 됨. {h:0,s:1,l:0.5}

## max == min 의 경우

회색 무채색의 경우일 가능성이 높음. 0,0,0 | 128,128,128
rgb가 모두 같으면 색상이 없다봄. 그래서 s는 0, h=의미없음 그래서 0으로 둠.
