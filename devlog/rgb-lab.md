# 왜 내가 뽑은 색상은 자꾸 탁해질까?..

> RGB로 이미지 추출을 해보며 배운 것들(?)

---

# 만들어보고 싶었던 것

## 이미지에 어울리는 배경 만들기

무언가 만들어보면서 공부할 주제를 찾다가  
Spotify의 앨범 커버 배경이 눈에 들어왔습니다.

> 이미지에서 색을 뽑으면  
> 배경색으로 쓸 만한 색을 만들 수 있지 않을까?

---

# 처음 떠올린 방법

이미지는 픽셀로 이루어져 있고, 픽셀은 색상 값을 가지고 있습니다.

그래서 처음에는 단순하게 생각했습니다.

> 이미지의 색상값을 가져와서 평균을내면 어울리는 배경색을 만들 수 있지 않을까?

---

# 그래서 RGB부터 보기 시작했습니다

이미지를 코드로 다룰 때 가장 먼저 생각나는 색상 값이 RGB였습니다.

```
pixel = rgb(r , g , b)
```

---

# RGB란?

Red · Green · Blue

RGB는 빨강, 초록, 파랑 빛의 세기로 색을 표현하는 방식입니다.

```
R = Red
G = Green
B = Blue
```

디스플레이는 이 세 가지 빛을 섞어서 우리가 보는 색을 표현합니다. ( 디스플레이, 컴퓨터 기준이더군요.. )

---

# RGB 값은 숫자로 다룰 수 있습니다

RGB 값은 보통 0부터 255 사이의 숫자로 표현됩니다.

```text
rgb(255, 0, 0)     = 빨강
rgb(0, 255, 0)     = 초록
rgb(0, 0, 255)     = 파랑

rgb(0, 0, 0)       = 검정
rgb(255, 255, 255) = 흰색
```

그래서 RGB는 코드에서 계산하기 쉬운 형태로 느껴졌습니다.

---

# 그래서 해본 첫 번째 방법

## 전체 RGB 평균

이미지의 모든 픽셀에서 RGB 값을 가져와서
각 채널별 평균을 구해봤습니다.

```
avgR = 모든 픽셀의 R 평균
avgG = 모든 픽셀의 G 평균
avgB = 모든 픽셀의 B 평균
```

이 평균값을 이미지의 대표색처럼 사용할 수 있지 않을까 생각했습니다.


# 왜 탁해졌을까?

## 평균은 모든 색을 섞어버립니다

이미지에는 포인트 컬러만 있는 것이 아닙니다.

포인트 컬러

```

- 배경색
- 그림자
- 흰색
- 검정
- 회색
  = 애매한 중간색

```

여러 색이 섞이면서 채도가 낮고 탁한 색이 나올 수 있었습니다.

---

# 많이 나온 색과 인상적인 색은 다릅니다

사람 눈에는 특정 포인트 컬러가 먼저 보일 수 있습니다.

하지만 픽셀 수로 보면 배경이나 어두운 영역이 더 많을 수 있습니다.

```

사람이 인상적으로 느끼는 색 ≠ 이미지에서 픽셀 수가 많은 색

```

---

# 여기서 생긴 문제

## 대표색 추출 ≠ 좋은 배경색 선택

RGB 평균값은 계산하기 쉽지만, 그 값이 바로 배경에 어울리는 색은 아니었습니다.

그래서 RGB 기반으로 몇 가지 보정을 시도해봤습니다.

---

# RGB로 해본 시도들

1. 이미지를 grid로 나누기
2. 너무 높은 RGB 값 절삭하기 ( 너무 밝거나 너무 어두운 부분을 때어버리는.. 느낌입니다 )
3. 결과 RGB 값에 +- 10 하기 ( 치우침에 따라서 )
4. 중앙 영역만 잘라서 분석하기

목표는 하나였습니다.

> 탁하게 나온 색을 조금이라도
> 배경색답게 만들어볼 수 없을까?

---

# 시도 1. Grid로 나누기

전체 이미지를 한 번에 평균내면 지역적인 색감이 사라지는 것처럼 보였습니다. ( 전체적인 부분이 너무 영향을 많이 받는게 문제가 아닌지.. 생각이 들더라고요.. )

그래서 이미지를 여러 영역으로 나누고, 각 영역별로 색을 계산해봤습니다.

```text
전체 이미지
→ 12칸 / 9칸 분할
→ 영역별 RGB 평균 계산
```

---

# Grid 방식 결과

좋았던 점

```
이미지 전체 평균보다
지역별 색 후보를 더 많이 볼 수 있었습니다
```

아쉬웠던 점

```
연산량이 늘었습니다
기대만큼 결과 차이가 크지는 않았습니다
결국 어떤 영역의 색을 쓸지 선택해야 했습니다
```

결론적으로,
영역을 나누는 것만으로는 해결되지 않았습니다.

---

# 시도 2. 높은 값 / 낮은 값 제외하기

너무 밝거나 너무 어두운 픽셀이
평균에 영향을 많이 주는 것 같았습니다.

그래서 일정 기준을 벗어나는 값을
제외하거나 줄여보려고 했습니다.

```
너무 밝은 값 제외
너무 어두운 값 제외
-> 나머지 값으로 평균 계산
```

---

# 값 제외 방식 결과

기대했던 점

```
극단적인 밝기 값의 영향을 줄이면
색이 조금 더 안정적으로 나오지 않을까?
```

실제 결과

```
색 정보가 같이 줄어드는 느낌이 있었습니다 ( 단순히 특징이 날라가버렸습니다. )
```

극단값을 줄이는 것만으로는
좋은 배경색을 만들기 어려웠습니다.

---

# 시도 3. RGB 값 보정하기

뽑힌 색이 너무 어둡거나 탁해 보여서,
결과 RGB 값에 일정 값을 더하거나 빼봤습니다. ( 총합이 어느정도 넘치면 줄이고 아닌경우 줄이고.. )

```
r = clamp(r + 10, 0, 255)
g = clamp(g + 10, 0, 255)
b = clamp(b + 10, 0, 255)
```

---

# RGB 보정 결과

좋았던 점

```
어두운 느낌이 조금 완화되는 경우가 있었습니다
생각보다 괜찮아 보이는 경우도 있었습니다
```

아쉬웠던 점

```
이미지마다 반응이 달랐습니다
어떤 경우에는 색이 떠 보였습니다 그리고 기준점을 찾기가 힘들었습니다.
근본적으로 좋은 색을 고른 것은 아니었습니다
```

단순 보정은 빠르지만,
일관된 규칙으로 쓰기에는 애매했습니다.

---

# 시도 4. 중앙 영역만 분석하기

이미지의 중요한 대상이
중앙에 있을 가능성이 높다고 생각했습니다.

그래서 전체 이미지가 아니라
중앙 영역만 잘라서 색을 분석해봤습니다.

전체 이미지
-> 중앙 영역 crop
-> 중앙 영역 RGB 평균 계산

---

# 중앙 crop 결과

괜찮게 나오는 경우도 있었습니다.

하지만 항상 맞지는 않았습니다.

```
중앙이 어두운 영역일 수 있었습니다
중앙이 배경일 수 있었습니다
이미지 구도에 따라 결과가 크게 달라졌습니다
```

이미지마다 구성이 달라서
안정적인 규칙으로 쓰기 어려웠습니다.

---

# 시행착오 정리

| 시도        | 기대               | 결과                      |
| ----------- | ------------------ | ------------------------- |
| RGB 평균    | 대표색을 쉽게 얻기 | 색이 탁해짐               |
| Grid 분할   | 지역 색감 반영     | 연산량 증가, 개선 폭 작음 |
| 극단값 제외 | 밝기 영향 줄이기   | 색감이 더 죽는 경우 있음  |
| RGB 보정    | 어두움/탁함 완화   | 이미지마다 결과가 다름    |
| 중앙 crop   | 주요 색 추출       | 이미지 구도에 의존        |

# RGB만으로 느낀 한계

RGB 값 조작은 직관적이었습니다.

하지만 원하는 결과를 안정적으로 만들기는 어려웠습니다.

```
평균을 내면 색이 섞입니다
값을 자르면 색 정보가 줄어듭니다
값을 더하면 밝아지지만 색이 뜰 수 있습니다
영역을 나눠도 최종 선택 문제는 남습니다
```

> 아마 이렇게 된 이유가 지금와서 보면 rgb는 사람이 보는 색상이 아니라 컴퓨터가 보는 색상.. 인 느낌이 강해서 그런게 아닌가 싶은 생각이 듭니다..

---

# 다시 정리하면

제가 원한 것은 단순한 평균색이 아니었습니다.

### 이미지의 분위기를 살릴 수 있는 배경색

```

픽셀 수가 많은 색 ≠ 사람이 인상적으로 느끼는 색

대표색 ≠ 배경에 쓰기 좋은 색
```

---

# 그래서 Lab를 보게 되었습니다

RGB에서 겪은 문제를 보면서, 색을 다른 방식으로 볼 필요가 있다고 느꼈습니다. ( 사람이 보는 색상에 가까운게 있는지 찾아보니.. Lab이 있었습니다.. )

```
RGB
-> 화면에 색을 표현하기 좋은 방식
-> 픽셀 값을 다루기 쉬움

Lab
-> 밝기와 색상 성분을 나눠서 보는 방식
-> 색의 차이를 다르게 볼 수 있음
```
